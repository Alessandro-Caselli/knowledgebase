## рџ”ђ Authentication Flow

1. User visits any protected route в†’ middleware redirects to `/login`
2. User clicks **Continue with Microsoft** в†’ `signIn('microsoft-entra-id')`
3. Azure AD handles OAuth2 + OpenID Connect
4. NextAuth receives callback with `profile.sub`, `user.email`, `user.name`
5. `signIn` callback:
   - Looks up user by email in SQLite
   - If new: creates record with `role = 'viewer'` (least privilege default)
   - Updates `last_login` and `azure_id`
   - Logs to `audit_log`
6. `jwt` callback: injects `userId` and `role` from DB into token
7. `session` callback: exposes `id` and `role` on `session.user`
8. Middleware checks `req.auth` on every request

## рџ›ЎпёЏ Permission System

### Hierarchy (highest wins)
```
Admin role в†’ full access, no checks needed
     в†“
User-specific permission (subject_type='user')
     в†“
Role-based permission (subject_type='role')
     в†“
Global permission (resource_type='global', resource_id='all')
     в†“
Default role rules: viewer=read, editor=read+write
```

### Permission Levels
| Level | Can do |
|---|---|
| `read` | View published documents |
| `write` | Create, edit documents |
| `admin` | Delete, manage permissions |

### Examples
- Grant user Alice `write` on the Engineering category в†’ she can create/edit docs there only
- Grant role `viewer` `read` on Finance globally в†’ all viewers see Finance
- Grant user Bob `admin` on document X в†’ he can delete that specific doc

## рџ”Ќ Search Implementation

- Uses **SQLite FTS5** (built-in, zero-dependency full-text search)
- Indexed fields: `title`, `content`, `summary`, `tags`
- Synced via three triggers: `AFTER INSERT`, `AFTER UPDATE`, `AFTER DELETE`
- Query uses prefix matching (`q*`) for real-time feel
- Returns `snippet()` with `<mark>` tags for highlight
- Filtered by category and respects user permission scope
- Debounced 300ms on the frontend

## вљ™пёЏ Setup Instructions

### 1. Clone & Install
```bash
git clone https://github.com/YOUR_ORG/knowledgebase.git
cd knowledgebase
npm install
```

### 2. Configure Azure AD
1. Go to [portal.azure.com](https://portal.azure.com)
2. **Azure Active Directory** в†’ **App Registrations** в†’ **New Registration**
3. Name: `KnowledgeBase`
4. Supported account types: **Single tenant** (or multitenant if needed)
5. Redirect URI (Web): `http://localhost:3000/api/auth/callback/microsoft-entra-id`
6. After creation, note:
   - **Application (client) ID** в†’ `AZURE_AD_CLIENT_ID`
   - **Directory (tenant) ID** в†’ `AZURE_AD_TENANT_ID`
7. **Certificates & secrets** в†’ **New client secret** в†’ copy the Value в†’ `AZURE_AD_CLIENT_SECRET`
8. For production, add: `https://yourdomain.com/api/auth/callback/microsoft-entra-id`

### 3. Environment Variables
```bash
cp .env.example .env.local
# Fill in AZURE_AD_CLIENT_ID, AZURE_AD_CLIENT_SECRET, AZURE_AD_TENANT_ID
# Generate secret: openssl rand -base64 32
```

### 4. Run
```bash
npm run dev
# App starts at http://localhost:3000
# SQLite DB auto-created at ./data/knowledgebase.db on first run
```

### 5. First Login
- Sign in with your Microsoft account
- The **first user** gets role `viewer` by default
- To make yourself admin: open `data/knowledgebase.db` with any SQLite client and run:
  ```sql
  UPDATE users SET role = 'admin' WHERE email = 'your@email.com';
  ```
- Or use the `sqlite3` CLI: `sqlite3 data/knowledgebase.db "UPDATE users SET role='admin' WHERE email='you@company.com';"`
